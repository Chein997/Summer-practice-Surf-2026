# Отфильтрованный набор тест-кейсов под текущую реализацию «Апекс»

Источник: исходный файл `apex-test-cases.md`.

## 1. Что фактически реализовано сейчас

На текущем этапе реализована не полноценная мобильная CMP-версия, а рабочий backend + статический web-demo:

| Компонент | Статус |
|---|---|
| PostgreSQL schema / migrations / seed | Реализовано |
| Backend health/readiness | Реализовано |
| Авторизация по телефону и dev SMS-коду `1111` | Реализовано |
| Список слотов | Реализовано |
| Детали слота | Реализовано |
| Создание брони | Реализовано |
| Просмотр моих броней | Реализовано |
| Просмотр деталей брони | Реализовано |
| Отмена брони клиентом | Реализовано |
| Регистрация / удаление push-token | Реализовано |
| Web-проверка через `web-demo/index.html` | Реализовано |
| Gradle bootstrap для запуска web-demo | Реализовано |
| Полноценный Compose Multiplatform client | Не реализовано |
| Админское подтверждение / отклонение брони | Не реализовано |
| Отмена заезда центром через API | Не реализовано |
| Реальная отправка push-уведомлений | Не реализовано |
| Push-напоминания за 24 часа / 2 часа | Не реализовано |
| Scheduler / background jobs | Не реализовано |

## 2. Итоговая стратегия фильтрации

Оставлены тест-кейсы, которые можно проверить одним из способов:

| Режим | Описание |
|---|---|
| `WEB_DEMO` | Можно проверить через `http://127.0.0.1:5173`. |
| `API` | Можно проверить через curl/Postman. |
| `API_WITH_DATA` | Реализовано, но нужны специальные seed/SQL-данные. |
| `SCOPE_NEGATIVE` | Проверка отсутствия не-MVP функциональности. |
| `BOOTSTRAP` | Проверка Gradle/web-demo bootstrap, добавлена под текущую реализацию. |

Исключены тест-кейсы, которые требуют полноценного mobile UI, админки, реальных push или фоновых задач.
