# Бизнес-требования

> Источник: `domain-description.md` — описание домена клиентского мобильного приложения картинг-центра «Апекс». Требования сформированы только из данных домена.

Приоритеты:

- **Must** — входит в MVP / обязательное бизнес-правило.
- **Won’t MVP** — явно отложено или вне скоупа MVP.

| ID | Формулировка | Источник | Приоритет |
|---|---|---|---|
| BR-001 | Система должна поддерживать клиентскую запись на групповые заезды в уличном картинг-центре через мобильное приложение. | domain-description.md, §1, §16 | Must |
| BR-002 | MVP должен включать просмотр доступных слотов. | domain-description.md, §2.1 | Must |
| BR-003 | MVP должен включать просмотр деталей слота. | domain-description.md, §2.1 | Must |
| BR-004 | MVP должен включать регистрацию / авторизацию клиента. | domain-description.md, §2.1 | Must |
| BR-005 | MVP должен включать бронирование места на заезд. | domain-description.md, §2.1 | Must |
| BR-006 | MVP должен включать просмотр клиентом своих броней. | domain-description.md, §2.1, §9 | Must |
| BR-007 | MVP должен включать отмену брони клиентом. | domain-description.md, §2.1, §10 | Must |
| BR-008 | MVP должен включать push-уведомления. | domain-description.md, §2.1, §12 | Must |
| BR-009 | MVP должен включать отображение отмены заезда центром. | domain-description.md, §2.1, §11 | Must |
| BR-010 | Мобильное приложение не должно управлять расписанием, маршалами, трассами, парком картов и административными действиями. | domain-description.md, §1, §15 | Must |
| BR-011 | Существующий бэкенд является источником истины по доступности мест, отсутствию двойных броней, статусам слотов, ручному подтверждению броней и отменам центра. | domain-description.md, §1, §15, §16 | Must |
| BR-012 | Для бронирования обязательны имя, телефон, email, возраст и согласие с правилами безопасности. | domain-description.md, §5 | Must |
| BR-013 | Минимальный возраст участия — 16 лет. | domain-description.md, §5, §14 | Must |
| BR-014 | Для клиентов младше 18 лет требуется согласие родителя или законного представителя. | domain-description.md, §5, §14 | Must |
| BR-015 | Клиенту по умолчанию должны показываться слоты на ближайшие 7 дней. | domain-description.md, §6, §14 | Must |
| BR-016 | Вместимость заезда ограничена количеством картов, типом заезда и ограничениями безопасности. | domain-description.md, §4, §14 | Must |
| BR-017 | Один аккаунт клиента может забронировать только одно место в одном заезде. | domain-description.md, §7, §14 | Must |
| BR-018 | Групповое бронирование в MVP не поддерживается. | domain-description.md, §7 | Won’t MVP |
| BR-019 | Бронь после создания не подтверждается автоматически и получает статус «Ожидает подтверждения». | domain-description.md, §7, §14 | Must |
| BR-020 | Подтверждение или отклонение брони выполняется администратором вручную в существующей админке. | domain-description.md, §7, §13 | Must |
| BR-021 | Автоматического тайм-аута подтверждения брони в MVP нет. | domain-description.md, §7, §14 | Won’t MVP |
| BR-022 | Клиент может отменить бронь только если до старта заезда осталось больше 1 часа. | domain-description.md, §10, §14 | Must |
| BR-023 | При успешной отмене клиентом место сразу освобождается. | domain-description.md, §10, §14 | Must |
| BR-024 | При отмене заезда центром бронь не удаляется, а получает статус «Отменена центром». | domain-description.md, §11, §14 | Must |
| BR-025 | Повторная запись на отменённый слот запрещена: отменённый слот показывается как недоступный для бронирования. | domain-description.md, §14 | Must |
| BR-026 | Фильтры по дате / времени / трассе, выбор экипировки, оценка маршала, расширенная программа лояльности и онлайн-оплата не входят в текущий MVP. | domain-description.md, §2.2 | Won’t MVP |
