# Use Cases

> Источник: `domain-description.md` — описание домена клиентского мобильного приложения картинг-центра «Апекс». Требования сформированы только из данных домена.

Приоритеты:

- **Must** — входит в MVP / обязательное бизнес-правило.
- **Won’t MVP** — явно отложено или вне скоупа MVP.

| ID | Формулировка | Источник | Приоритет |
|---|---|---|---|
| UC-001 | Авторизация клиента: клиент вводит номер телефона, подтверждает вход SMS-кодом и получает доступ к действиям приложения. | domain-description.md, §5, §13 | Must |
| UC-002 | Просмотр списка слотов: клиент открывает приложение, приложение получает из API слоты на ближайшие 7 дней и отображает их список. | domain-description.md, §6, §13 | Must |
| UC-003 | Просмотр пустого состояния: если расписания на ближайшие дни нет, приложение показывает сообщение об отсутствии доступных заездов. | domain-description.md, §6 | Must |
| UC-004 | Просмотр деталей слота: клиент выбирает слот и видит дату, время, длительность, трассу, уровень, места, цену, маршала, адрес, место сбора, правила безопасности, условия отмены и кнопку бронирования. | domain-description.md, §6.2, §13 | Must |
| UC-005 | Бронирование места: клиент нажимает «Забронировать», приложение отправляет запрос в API, бэкенд проверяет доступность и создаёт бронь либо возвращает отказ. | domain-description.md, §7, §13 | Must |
| UC-006 | Создание брони в ожидании: при успешном создании бронь получает статус «Ожидает подтверждения», который отображается клиенту. | domain-description.md, §7, §8, §13 | Must |
| UC-007 | Подтверждение брони администратором: администратор подтверждает бронь в существующей админке, бронь становится активной, клиент получает push-уведомление. | domain-description.md, §7, §12, §13 | Must |
| UC-008 | Отклонение брони администратором: администратор отклоняет бронь, она получает статус «Отклонена центром», клиент получает push-уведомление. | domain-description.md, §7, §12, §13 | Must |
| UC-009 | Просмотр своих броней: клиент открывает раздел «Мои брони» и видит собственные брони с актуальными статусами. | domain-description.md, §9 | Must |
| UC-010 | Отмена брони клиентом: клиент отменяет активную или ожидающую бронь более чем за 1 час до старта, после успешной отмены место освобождается. | domain-description.md, §10, §13 | Must |
| UC-011 | Запрет отмены брони: если до старта осталось 1 час или меньше, приложение запрещает отмену через приложение. | domain-description.md, §10 | Must |
| UC-012 | Отмена заезда центром: центр отменяет заезд, бронь получает статус «Отменена центром», клиент видит причину, дату и время заезда и кнопку «Выбрать другой заезд». | domain-description.md, §11, §13 | Must |
| UC-013 | Получение напоминания о заезде: клиент получает push-напоминание за 24 часа и за 2 часа до старта. | domain-description.md, §12 | Must |
| UC-014 | Обработка недоступного бронирования: приложение отображает слот без возможности бронирования, если мест нет или слот отменён. | domain-description.md, §6.1, §15 | Must |
| UC-015 | Обработка отказа API: приложение обрабатывает отказ при отсутствии мест, отменённом слоте, отклонении брони или недоступности действия. | domain-description.md, §15 | Must |
